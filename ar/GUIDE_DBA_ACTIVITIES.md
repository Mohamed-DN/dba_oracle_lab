# دليل: أنشطة مدير قاعدة البيانات الأساسية — الدفعات، AWR، الترقيع، Data Pump، والأمان

> **الهدف**: يغطي هذا الدليل أنشطة مدير قاعدة البيانات (DBA) اليومية التي كانت مفقودة في المختبر: وظائف الدفعات (Batch jobs)، تحليل الأداء (AWR/ADDM/ASH)، عمليات الترقيع (Patching)، استيراد وتصدير البيانات عبر Data Pump، تعزيز الأمان، وإدارة مساحات الجداول (Tablespaces).
>
> يتضمن كل قسم أوامر جاهزة للتطبيق في المختبر وملاحظات لبيئات الإنتاج الحقيقية.

---

## 1. وظائف الدفعات (Batch Jobs) باستخدام DBMS_SCHEDULER

نظام `DBMS_SCHEDULER` هو نظام الجدولة المتطور في أوراكل، والذي حل محل `DBMS_JOB` القديم.

- **المكونات الرئيسية**:
  - **PROGRAM**: ماذا سننفذ (PL/SQL، سكربت، أو برنامج).
  - **SCHEDULE**: متى سننفذ (توقيت مشابه لـ cron).
  - **JOB**: يجمع بين البرنامج والجدول الزمني.

### 1.2 وظيفة جمع الإحصائيات (مهمة للمحسن/Optimizer)
```sql
BEGIN
  DBMS_SCHEDULER.CREATE_JOB(
    job_name        => 'JOB_GATHER_STATS',
    job_type        => 'PLSQL_BLOCK',
    job_action      => 'BEGIN DBMS_STATS.GATHER_DATABASE_STATS(options => ''GATHER STALE''); END;',
    start_date      => SYSTIMESTAMP,
    repeat_interval => 'FREQ=DAILY; BYHOUR=2',
    enabled         => TRUE,
    comments        => 'Nightly statistics gathering'
  );
END;
/
```

## 2. تحليل الأداء: AWR, ADDM و ASH

- **AWR (Automatic Workload Repository)**: يأخذ لقطات (Snapshots) لنشاط قاعدة البيانات كل ساعة ويحتفظ بها لمدة 8 أيام.
- **ADDM**: "طبيب" قاعدة البيانات الذي يحلل لقطات AWR ويقدم توصيات (مثل "أضف ذاكرة RAM").
- **ASH**: يراقب الجلسات النشطة كل ثانية، وهو مثالي لمعرفة "ماذا كان يحدث قبل 5 دقائق؟".

### توليد تقرير AWR (HTML):
```sql
-- 1. عرض اللقطات المتاحة
SELECT snap_id, begin_interval_time FROM dba_hist_snapshot ORDER BY snap_id DESC;

-- 2. تشغيل السكربت (سيطلب منك رقم لقطة البداية والنهاية)
@$ORACLE_HOME/rdbms/admin/awrrpt.sql
```

## 3. ترقيع أوراكل (Oracle Patching)

عملية الترقيع هي تحديث البرنامج لسد الثغرات الأمنية أو إصلاح الأخطاء (Bugs).

1. **التحضير**: تحميل الـ Patch من My Oracle Support وتحديث أداة **OPatch**.
2. **التطبيق في RAC**: نستخدم `opatchauto` لضمان استمرارية العمل (تحويل العقد واحدة تلو الأخرى - Rolling Patch).
3. **تحديث القاموس**: تشغيل `datapatch` لتطبيق التغييرات داخل قاعدة البيانات.

---

## 4. استيراد وتصدير البيانات (Data Pump)

نستخدم `expdp` للتصدير و `impdp` للاستيراد. تذكر أن ملفات الـ `.dmp` تُنشأ دائماً على الخادم (Server) داخل مسار يُعرف بـ `DIRECTORY` في أوراكل.

### مثال تصدير مستخدم (Schema):
```bash
expdp system/pass schemas=HR directory=DPUMP_DIR dumpfile=hr.dmp logfile=hr_exp.log
```

---

## 5. تعزيز الأمان (Security Hardening)

- **ملفات التعريف (Profiles)**: التحكم في قوة كلمة المرور، عدد محاولات الدخول الفاشلة، ومدة صلاحية الكلمة.
- **التدقيق الموحد (Unified Auditing)**: مراقبة من فعل ماذا ومتى (مثل محاولات الدخول الفاشلة أو تعديل كشوفات الرواتب).
- **التشفير (TDE)**: تشفير البيانات على القرص لضمان عدم قراءتها في حال سرقة ملفات البيانات.

## قائمة المهام اليومية لمدير قاعدة البيانات (Checklist)

| المهمة | التوقيت | الهدف |
|---|---|---|
| فحص الـ Alert Log | صباحاً | البحث عن أخطاء ORA- |
| فحص المساحة (Tablespaces) | صباحاً | التأكد من وجود مساحة كافية (>15%) |
| التأكد من نجاح النسخ الاحتياطي | صباحاً | ضمان القدرة على الاستعادة |
| مراجعة توصيات ADDM | أسبوعياً | تحسين الأداء |
| اختبار استعادة RMAN | شهرياً | التأكد من سلامة النسخ تقنياً |

# دليل: إدارة CDB/PDB، المستخدمين و EM Express

> يغطي هذا الدليل 3 مجالات أساسية لمدير قاعدة بيانات أوراكل 19c، والتي غالباً ما يتم تجاهلها في مختبرات RAC:
> بنية الحاويات المتعددة (CDB/PDB)، إدارة المستخدمين والصلاحيات، وأداة Enterprise Manager Express.

---

## الجزء 1: بنية الحاويات المتعددة (Multitenant Architecture)

### ما هي ولماذا توجد؟
قبل إصدار أوراكل 12c، كانت كل قاعدة بيانات مستقلة تماماً (بنسخة منفصلة من الذاكرة والقاموس). إذا كان لديك 10 تطبيقات، كنت تحتاج لـ 10 قواعد بيانات (هدر في الذاكرة والقرص).

**أوراكل 12c وما بعدها قدمت مفهوم Multitenant:**

```
┌───────────────────────────────────────────────────────────────────┐
│                بنية CDB/PDB (الإصدارات الحديثة)                   │
│                                                                   │
│               نسخة واحدة (Instance) تشارك الذاكرة                 │
│               ┌──────────────────────────────────┐                │
│               │  SGA (مشاركة بين الجميع)          │                │
│               └──────────────┬───────────────────┘                │
│                              │                                    │
│   ┌──────────────────────────┴───────────────────────────┐        │
│   │                          CDB$ROOT                     │        │
│   │                    (القاموس الرئيسي)                  │        │
│   └──────┬──────────┬──────────┬─────────────────────────┘        │
│          │          │          │                                   │
│   ┌──────┴──┐ ┌─────┴───┐ ┌───┴─────┐                            │
│   │PDB$SEED │ │  PDB_A  │ │  PDB_B  │     ← كل قاعدة PDB لها     │
│   │(قالب    │ │  تطبيق أ │ │  تطبيق ب │        ملفاتها الخاصة      │
│   │ فارغ)   │ │         │ │         │                            │
│   └─────────┘ └─────────┘ └─────────┘                            │
└───────────────────────────────────────────────────────────────────┘
```

### العمليات الأساسية التي يجب معرفتها:

- **التنقل بين الحاويات**:
```sql
SHOW CON_NAME; -- معرفة الحاوية الحالية
SHOW PDBS;     -- رؤية جميع القواعد الفرعية PDBs
ALTER SESSION SET CONTAINER = PDB1; -- الانتقال إلى PDB1
```

- **إنشاء/حذف PDB**:
```sql
CREATE PLUGGABLE DATABASE PDB2 FROM PDB1; -- استنساخ قاعدة
ALTER PLUGGABLE DATABASE PDB2 OPEN;        -- فتح القاعدة للعمل
```

## الجزء 2: إدارة المستخدمين والأدوار (Roles)

### أنواع المستخدمين في أوراكل:
1. **SYS (SYSDBA)**: المالك الفعلي لقاموس البيانات. يمتلك كامل الصلاحيات (بدء/إيقاف/استعادة). لا يستخدم للأعمال اليومية.
2. **SYSTEM**: مدير للنظام ولكن ليس مالكاً للقاموس. يستخدم للأعمال الإدارية اليومية.
3. **المستخدمون المخصصون**: مثل `dba_admin` أو مستخدمي التطبيقات.

### أفضل ممارسات إدارة المستخدمين:
- **مبدأ أقل صلاحية**: لا تمنح صلاحيات DBA لمستخدم تطبيق. أنشئ دوراً (Role) يحتوي فقط على ما يحتاجه المستخدم.
- **تأمين كلمة المرور (Profiles)**:
```sql
CREATE PROFILE secure_profile LIMIT
  FAILED_LOGIN_ATTEMPTS 5         -- القفل بعد 5 محاولات فاشلة
  PASSWORD_LIFE_TIME 90           -- تنتهي الصلاحية كل 90 يوماً
  PASSWORD_LOCK_TIME 1/24;        -- القفل لمدة ساعة واحدة
```

## الجزء 3: Enterprise Manager Database Express (EM Express)

هي واجهة ويب مدمجة في أوراكل 19c لا تتطلب أي تثبيت إضافي. تعمل على المنفذ 5500 بشكل افتراضي.

### تفعيل الواجهة في المختبر:
```sql
EXEC DBMS_XDB_CONFIG.SETHTTPSPORT(5500);
```
بعد ذلك يمكنك الوصول إليها عبر المتصفح: `https://rac1:5500/em/`

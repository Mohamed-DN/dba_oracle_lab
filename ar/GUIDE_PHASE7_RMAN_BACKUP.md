# المرحلة 7: استراتيجية النسخ الاحتياطي RMAN لجميع قواعد البيانات

> النسخ الاحتياطي هو خط دفاعك الأخير. مهما كانت حلول التوافر العالي (RAC, Data Guard, GoldenGate) متطورة، فإن الخطأ البشري قد يحذف بيانات لا يمكن استعادتها إلا عبر RMAN.

---

## 7.1 استراتيجية النسخ الاحتياطي

يتم توزيع مهام النسخ الاحتياطي لتقليل الحمل على قاعدة البيانات الأساسية:
1. **الأساسي (RACDB)**: نسخ احتياطي لسجلات الأرشيف (Archivelog) كل ساعتين + مستوى 1 خفيف يومياً.
2. **الاحتياطي (RACDB_STBY)**: النسخ الاحتياطي **الرئيسي** (مستوى 0 ومستوى 1). بما أن الموارد لا تستهلكها التطبيقات هنا، فمن الأفضل استغلالها للنسخ.
3. **المستهدف (dbtarget)**: نسخ احتياطي مستقل على القرص المحلي.

---

## 7.2 التكوين الأساسي لـ RMAN

نفذ هذه الإعدادات على جميع قواعد البيانات:
```rman
-- نافذة الاستعادة لمدة 7 أيام
CONFIGURE RETENTION POLICY TO RECOVERY WINDOW OF 7 DAYS;

-- النسخ الاحتياطي التلقائي لملف التحكم و SPFILE
CONFIGURE CONTROLFILE AUTOBACKUP ON;
CONFIGURE CONTROLFILE AUTOBACKUP FORMAT FOR DEVICE TYPE DISK TO '+FRA/%F';

-- تفعيل التوازي (Parallelism) والضغط (Compression)
CONFIGURE DEVICE TYPE DISK PARALLELISM 2 BACKUP TYPE TO COMPRESSED BACKUPSET;
CONFIGURE COMPRESSION ALGORITHM 'MEDIUM';

-- تحسين النسخ الاحتياطي (تخطي الملفات التي لم تتغير)
CONFIGURE BACKUP OPTIMIZATION ON;
```

---

## 7.3 تتبع تغيير الكتل (Block Change Tracking - BCT)

تفعيل BCT يجعل النسخ التزايدي أسرع بـ 10 إلى 100 مرة:
```sql
ALTER DATABASE ENABLE BLOCK CHANGE TRACKING USING FILE '+DATA/RACDB/bct_racdb.dbf';
```

---

## 7.4 سكربتات النسخ الاحتياطي (على الاحتياطي)

### المستوى 0 (كامل) - أسبوعياً
يتم نسخ كامل قاعدة البيانات مع سجلات الأرشيف وحذف السجلات التي تم نسخها لتوفير المساحة في الـ FRA.

### المستوى 1 (تزايدي) - يومياً
نسخ الكتل التي تغيرت فقط منذ آخر نسخة، وهو أسرع بكثير ويوفر المساحة.

---

## 7.10 فحص الحالة (Health Check) والصيانة التلقائية

لا تكتمل وظيفة الـ DBA بدون مراقبة يومية. تأكد من:
1. **جمع الإحصائيات (Stats)**: محرك أوراكل يحتاج إحصائيات حديثة للجداول لاختيار أفضل خطة تنفيذ (Execution Plan).
2. **فحص المساحة**: مراقبة الـ Tablespaces ومجموعات أقراص ASM.
3. **سجل التنبيهات (Alert Log)**: البحث عن أخطاء `ORA-`.
4. **اختبار الاستعادة**: النسخة التي لم يتم اختبار استعادتها لا تعتبر نسخة! استخدم `RESTORE DATABASE VALIDATE` دورياً.

---
## 🎉 تهانينا! لقد أتممت المختبر بالكامل.

لقد قمت ببناء بيئة أوراكل متكاملة واحترافية تشمل:
1. **RAC**: توافر عالي محلي.
2. **Data Guard**: حماية من الكوارث (DR).
3. **GoldenGate**: تكرار منطقي للبيانات نحو السحابة.
4. **RMAN**: استراتيجية نسخ واستعادة متكاملة.
5. **إدارة الأداء والصيانة**: فحوصات دورية وإحصائيات.

# Guida Package Monitor DDL — Audit Operativo Oggetti

## Obiettivo

Implementare un monitor DDL robusto (tabella log + package + trigger) per tracciare modifiche a oggetti Oracle in modo centralizzato, con retention e query operative.

## Teoria

Un audit DDL custom è utile quando vuoi visibilità immediata su:

- `CREATE`, `ALTER`, `DROP` oggetti
- schema coinvolto
- SQL text e contesto sessione

Approccio consigliato:

1. tabella `DDL_AUDIT_LOG`
2. package `PKG_MONIT_DDL` per inserimento e purge
3. trigger `AFTER DDL ON DATABASE` con `AUTONOMOUS_TRANSACTION`

## Procedura Operativa

### 1) Crea tabella log

```sql
CREATE TABLE dba_tools.ddl_audit_log (
  id              NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  event_time      TIMESTAMP DEFAULT SYSTIMESTAMP NOT NULL,
  db_user         VARCHAR2(128),
  os_user         VARCHAR2(128),
  host            VARCHAR2(256),
  ip_address      VARCHAR2(64),
  module          VARCHAR2(64),
  object_owner    VARCHAR2(128),
  object_name     VARCHAR2(128),
  object_type     VARCHAR2(64),
  event_name      VARCHAR2(64),
  sql_text        CLOB
);
```

### 2) Crea package di monitor

```sql
CREATE OR REPLACE PACKAGE dba_tools.pkg_monit_ddl AS
  PROCEDURE log_event;
  PROCEDURE purge_old(p_days NUMBER DEFAULT 90);
END pkg_monit_ddl;
/

CREATE OR REPLACE PACKAGE BODY dba_tools.pkg_monit_ddl AS
  PROCEDURE log_event IS
    PRAGMA AUTONOMOUS_TRANSACTION;
    l_sql ORA_NAME_LIST_T;
    l_n   PLS_INTEGER;
    l_txt CLOB;
  BEGIN
    l_n := ora_sql_txt(l_sql);
    FOR i IN 1 .. l_n LOOP
      l_txt := l_txt || l_sql(i);
    END LOOP;

    INSERT INTO dba_tools.ddl_audit_log(
      db_user, os_user, host, ip_address, module,
      object_owner, object_name, object_type, event_name, sql_text
    ) VALUES (
      SYS_CONTEXT('USERENV','SESSION_USER'),
      SYS_CONTEXT('USERENV','OS_USER'),
      SYS_CONTEXT('USERENV','HOST'),
      SYS_CONTEXT('USERENV','IP_ADDRESS'),
      SYS_CONTEXT('USERENV','MODULE'),
      ORA_DICT_OBJ_OWNER,
      ORA_DICT_OBJ_NAME,
      ORA_DICT_OBJ_TYPE,
      ORA_SYSEVENT,
      l_txt
    );
    COMMIT;
  EXCEPTION
    WHEN OTHERS THEN
      ROLLBACK;
  END log_event;

  PROCEDURE purge_old(p_days NUMBER DEFAULT 90) IS
  BEGIN
    DELETE FROM dba_tools.ddl_audit_log
    WHERE event_time < SYSTIMESTAMP - NUMTODSINTERVAL(p_days, 'DAY');
    COMMIT;
  END purge_old;
END pkg_monit_ddl;
/
```

### 3) Crea trigger DDL globale

```sql
CREATE OR REPLACE TRIGGER dba_tools.trg_monit_ddl
AFTER DDL ON DATABASE
BEGIN
  dba_tools.pkg_monit_ddl.log_event;
END;
/
```

### 4) Crea job di retention

```sql
BEGIN
  DBMS_SCHEDULER.CREATE_JOB(
    job_name        => 'DBA_TOOLS.JOB_PURGE_DDL_AUDIT',
    job_type        => 'PLSQL_BLOCK',
    job_action      => 'BEGIN dba_tools.pkg_monit_ddl.purge_old(90); END;',
    repeat_interval => 'FREQ=DAILY;BYHOUR=3;BYMINUTE=0;BYSECOND=0',
    enabled         => TRUE
  );
END;
/
```

## Esempio

```sql
ALTER TABLE hr.employees ADD (note VARCHAR2(200));

SELECT event_time, db_user, object_owner, object_name, event_name
FROM dba_tools.ddl_audit_log
ORDER BY id DESC FETCH FIRST 20 ROWS ONLY;
```

## Validazione Finale

```sql
SELECT owner, trigger_name, status
FROM dba_triggers
WHERE trigger_name = 'TRG_MONIT_DDL';
```

```sql
SELECT COUNT(*) AS rows_logged
FROM dba_tools.ddl_audit_log;
```

Atteso:

- Trigger `ENABLED`
- Log popolato dopo operazioni DDL
- Job purge schedulato e attivo

## Troubleshooting Rapido

- **Nessuna riga nel log**: verifica owner trigger/package e grant su tabella log.
- **Errori su trigger DDL**: controlla oggetti INVALID (`dba_objects`).
- **Crescita log eccessiva**: riduci retention (`purge_old`) o filtra eventi.

## Note operative

- Esegui installazione in schema tecnico dedicato (`DBA_TOOLS`).
- In produzione valuta integrazione con Unified Auditing/SIEM.
- Evita audit di SQL text se contiene dati sensibili non necessari.

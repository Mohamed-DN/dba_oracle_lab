# Community Onboarding Path

## Percorso contributor (30 giorni)

### Giorni 1-7: Setup e primo contributo
- Leggere `CONTRIBUTING.md`
- Aprire una issue con template `good first issue`
- Eseguire un fix docs piccolo con PR

### Giorni 8-15: Contributo tecnico guidato
- Scegliere task runbook/script a basso rischio
- Allegare evidenze validazione
- Completare checklist didattica standard

### Giorni 16-30: Contributo avanzato
- Proporre miglioramento automazione/monitoring
- Aggiornare KPI/compatibilità se impattati
- Fare review di almeno 1 PR altrui

## Roadmap didattica trimestrale

- Q2: fondamenti + runbook core
- Q3: HA/DR + performance tuning
- Q4: governance, security, automation avanzata
